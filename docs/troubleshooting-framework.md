# My Troubleshooting Framework

## Step 1: Define the issue
- What exactly is happening?
- What should happen instead?
- Who is affected?

## Step 2: Classify the issue
- User error
- Permission/configuration issue
- Data issue
- Integration/API issue
- System defect
- Environment issue

## Step 3: Reproduce and validate
- Can the issue be reproduced?
- Is it consistent or intermittent?
- Is it isolated or widespread?

## Step 4: Investigate evidence
- SQL/data checks
- API requests/responses
- Logs/errors
- Workflow/permissions
- Browser DevTools

## Step 5: Decide next action
- Resolve directly
- Provide workaround
- Escalate to engineering/vendor
- Document for future reference
